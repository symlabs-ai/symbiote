# Symbiote v0.6.0 — Release Report

**Data:** 2026-05-26
**Tipo:** Minor (0.5.0 → 0.6.0) — aditiva, sem breaking changes
**Para clientes:** SymbiOS, hosts embedados, e qualquer aplicação que usa o kernel Symbiote
**CHANGELOG completo:** [`CHANGELOG.md`](../../CHANGELOG.md#v060---2026-05-26)

---

## TL;DR

O Symbiote agora tem **websearch, extração de conteúdo e navegação em browsers** como tools opt-in. Tudo via o subpackage novo `symbiote.browser`. Quem não chamar `register()`, **nada muda** — `import symbiote` continua leve e a API atual fica intocada.

```python
from symbiote.browser import register

register(
    kernel,
    search_backend="brave",
    extract_backend=["forge_scraper", "firecrawl"],
    browser_backend="chromium",
    policy={"blocklist": ["*.ads.com"]},
)
# Pronto. 8 tools novas registradas no ToolGateway, Symbiota pode usar.
```

## O que entrou

### Tools novas no kernel

| Tool ID | O que faz | Backend | Custo |
|---|---|---|---|
| `web_search` | Busca textual na web (top N hits com title/URL/snippet) | Brave Search via SymGateway | $0.003/query |
| `web_extract` | Extrai conteúdo limpo de uma URL específica (texto + metadados) | forge_scraper → fallback Firecrawl | $0 com forge_scraper |
| `web_crawl` | Crawl direcionado de um domínio por instrução natural | Firecrawl via SymGateway | aguarda seed DevOps |
| `browser_navigate` | Abre URL em sessão Chromium isolada | Playwright local | $0 |
| `browser_snapshot` | Accessibility tree da página com refs `@e1`, `@e2`… | Playwright local | $0 |
| `browser_click` | Clica elemento por ref | Playwright local | $0 |
| `browser_fill` | Preenche input por ref | Playwright local | $0 |
| `browser_screenshot` | PNG base64 da viewport ou página inteira | Playwright local | $0 |
| `browser_wait_for` | Espera texto aparecer (JS async) | Playwright local | $0 |
| `browser_close` | Encerra sessão | Playwright local | $0 |

### Segurança

- **`WebsitePolicy`** — blocklist/allowlist por domínio com wildcards (`*.ads.com`) + TTL cache. Aplica em `browser_navigate` antes de qualquer I/O.
- **SSRF guard** — `validate_url` roda em todo `navigate` / `extract` / `crawl` antes da requisição.
- **PolicyGate + audit_log** — toda chamada passa pelo PolicyGate existente; cada execução vira linha em `audit_log` com `request_id`, `cost_usd`, `elapsed_ms`.

## Instalação

```bash
# Apenas web_search (sem deps extra — usa httpx, já em dependencies)
pip install symbiote  # nada de novo

# Para web_extract via forge_scraper
pip install "symbiote[extract]"

# Para browser automation
pip install "symbiote[browser]"
playwright install chromium     # uma vez, ~170MB

# Tudo
pip install "symbiote[extract,browser]"
playwright install chromium
```

`symbiote.browser` é **opt-in**: enquanto você não chamar `register()`, o kernel se comporta como na v0.5.0. Sem mudanças de import, sem aumento de tempo de startup, sem deps extra carregadas.

## Quickstart

### Web search (5 linhas)

```python
from symbiote.core.kernel import SymbioteKernel
from symbiote.config.models import KernelConfig
from symbiote.browser import register

kernel = SymbioteKernel(KernelConfig(db_path="data/db.sqlite"), llm=your_llm)
register(kernel, search_backend="brave")

bot = kernel.create_symbiote(name="Researcher", role="web researcher")
kernel.environment.configure(symbiote_id=bot.id, tools=["web_search"])
session = kernel.start_session(bot.id, goal="research")
print(kernel.message(session.id, "What's the current Python version?"))
```

O Symbiota decide chamar `web_search`, recebe os hits, e responde.

### Web extract (free via forge_scraper)

```python
register(kernel, extract_backend="forge_scraper")
# Symbiota pode usar web_extract pra ler artigos limpos:
#   web_extract({"url": "https://docs.python.org/3/tutorial/index.html"})
#   → {url, title, content (markdown), language, platform, metadata}
```

Cobertura platform-aware do `forge_scraper`:

| Domínio | Comportamento |
|---|---|
| YouTube (`youtube.com`, `youtu.be`) | Retorna transcript completo + metadata do vídeo |
| Reddit (`reddit.com`) | Retorna post + top comments |
| Twitter/X (`x.com`, `twitter.com`) | Conteúdo do tweet |
| Instagram | Conteúdo do post (precisa `[instagram]` extra com `instaloader`) |
| Sites genéricos | Markdown limpo via trafilatura |

### Web extract com fallback (recomendado em produção)

```python
register(kernel, extract_backend=["forge_scraper", "firecrawl"])
# Symbiota tenta forge_scraper primeiro. Se falhar ou retornar vazio,
# o handler automaticamente tenta Firecrawl via SymGateway.
# O campo result["extracted_by"] indica qual provider venceu.
```

### Navegação real (Chromium)

```python
from symbiote.browser.config import BrowserOptions

register(
    kernel,
    browser_backend="chromium",
    browser_options=BrowserOptions(headed=False, timeout_ms=20_000),
)
# Symbiota pode:
#   browser_navigate({"url": "https://...", "task_id": "t1"})
#   browser_snapshot({"task_id": "t1"})  → aria tree com @e1, @e2...
#   browser_click({"ref": "@e3", "task_id": "t1"})
#   browser_screenshot({"task_id": "t1", "full_page": true})
#   browser_close({"task_id": "t1"})
```

### Setup recomendado pra SymbiOS

```python
register(
    kernel,
    search_backend="brave",
    extract_backend=["forge_scraper", "firecrawl"],
    browser_backend="chromium",
    policy={
        "blocklist": ["*.ads.com", "doubleclick.net", "facebook.com"],
    },
)
```

8 tools ativas, billing centralizado via SymGateway, policy de domínio aplicada, fallback configurado.

## Cost cheat-sheet

| Operação | Provider primário | Custo |
|---|---|---|
| `web_search` | Brave via SymGateway | $0.003/query |
| `web_extract` (URL pública sem proxy) | forge_scraper local | $0 |
| `web_extract` (URL atrás de Cloudflare/etc) | forge_scraper via SymGateway proxy | custo do proxy SymGateway |
| `web_extract` fallback | Firecrawl via SymGateway | custo Firecrawl |
| `web_crawl` | Firecrawl via SymGateway | custo Firecrawl × páginas |
| `browser_*` (Chromium local) | seu próprio CPU/RAM | $0 |

Brave free tier: $5/mês de créditos prepagos. Para 1k–10k queries/mês, Symbiote sai mais barato que qualquer alternativa que considerei (Tavily/Exa/Firecrawl-direct).

## Setup do ambiente

### Credenciais

Você **não precisa pedir credenciais ao usuário final**. As que existem para search/extract estão centralizadas no SymGateway:

- `SYMGATEWAY_API_KEY` (já em uso pelo LLM)
- `SYMGATEWAY_BASE_URL` (já em uso pelo LLM)

Quando precisar de credenciais novas (Firecrawl seed, Browserbase, etc), abra `/ask devops` — DevOps coloca no SymVault e expõe via proxy do gateway. Você nunca tem que mexer com keys diretamente.

### Variáveis de ambiente

| Variável | Obrigatória pra | Onde |
|---|---|---|
| `SYMGATEWAY_API_KEY` | `web_search`, fallback Firecrawl | `.env` do host |
| `SYMGATEWAY_BASE_URL` | idem | `.env` do host |

Se já está rodando Symbiote com LLM via SymGateway, **não precisa adicionar nada** — as mesmas vars cobrem search e extract.

## Limitações conhecidas

| Item | Status | Workaround |
|---|---|---|
| **`web_crawl` precisa de seed Firecrawl no SymGateway** | DevOps pendente — script template em `sym_gateway/scripts/seed_brave_search.py` | Usar `browser_navigate` + `browser_snapshot` em loop até DevOps liberar |
| **Stealth opt-in (anti-fingerprinting)** | Não implementado v0.6.0 | Sites com anti-bot agressivo: fallback para Firecrawl no chain de extract |
| **Cloud browsers (Browserbase / Browser Use)** | Não implementado v0.6.0 | Use Chromium local; cloud entra em v0.6.1+ se houver demanda |
| **DuckDuckGo HTML scraping como alternativa free** | Investigado, inviabilizado | Brave free tier ($5/mês) cobre dev; ver [Fase 7 do plano](../integrations/symbiote-browser.md) pra a matriz completa de evidências |
| **`forge_scraper` requer sub-extras `[article,transcript,sofascore]`** | Bug de packaging upstream (`__init__.py` eager-import) | Já tratado: `pip install "symbiote[extract]"` puxa o conjunto correto automaticamente |

## Migração da v0.5.x

**Nenhuma ação necessária.** Atualizar a dependência:

```bash
pip install -U symbiote
```

Se você quer ativar as tools novas, adiciona a linha de `register()` onde inicializa o kernel. Senão, deixa quieto e tudo continua igual.
